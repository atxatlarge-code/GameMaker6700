## 2023-10-27 - A* Pathfinding Open Set Performance
**Learning:** The A* pathfinding open set in `src/pathfinder.js` was using a standard array with binary search and `array.splice()` to insert elements. While binary search finds the index in O(log N) time, `array.splice()` shifts elements, resulting in an O(N) insertion bottleneck as the open set grows. Furthermore, the original array binary search naturally maintained strict LIFO tie-breaking for equal fScores due to inserting at the *first* matching index.
**Action:** Replaced the array with a custom Min-Heap implementation to achieve true O(log N) insertion/extraction. Crucially, strict LIFO tie-breaking was intentionally preserved in the heap's compare function by tracking insertion order with a counter (higher ID = inserted later = higher priority for identical fScores). This maintained perfect functional equivalence (identical path lengths and iterations) while vastly speeding up execution.
